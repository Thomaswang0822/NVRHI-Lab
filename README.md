# NVRHI-Lab

A graphics learning laboratory using NVRHI to understand modern graphics API fundamentals (D3D11, D3D12, Vulkan).

## Overview

NVRHI-Lab provides an interactive testbed for exploring graphics API concepts through NVRHI's abstraction layer. The application follows an AimLab-like design with targets and environments, enabling hands-on experimentation with resources, pipelines, bindings, and more.

**Key Principles:**
- **Comprehensive coverage**: Implement all NVRHI-supported features (8 primitive types, various texture formats, etc.)
- **Backend-agnostic learning**: Focus on NVRHI API concepts, not backend implementation details
- **Incremental complexity**: Start with a simple colored triangle, progress to interactive 3D scenes

## Quick Start

### Prerequisites
- Windows (x64)
- Visual Studio 2019 or later
- CMake 3.20 or later (VS 2026 solution which I'm using can be generated by CMake >= 4.1.1)
- Git submodules for third-party libraries

### Build

```bash
git submodule update --init --recursive
cd build
cmake .. -G "Visual Studio 17 2026" -A x64
```

Open `NVRHI-Lab.slnx` in Visual Studio and build.

### Run

Choose backend (DX11/DX12/Vulkan) on startup, then experiment with different rendering scenarios through the menu.

## Tech Stack

- **Language**: C++20
- **Build System**: CMake
- **Graphics Abstraction**: NVRHI (DX11, DX12, Vulkan)
- **GUI Framework**: wxWidgets
- **Shaders**: HLSL
